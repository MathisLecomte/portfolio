The Linux Quizz

December 27, 2020

The Linux Quizz is a 12 question quiz about the Linux operating system and its many applications.

The goal of this quiz is above all to handle a MySQL database from a PHP code, allowing a authentication by login pattern, management of a session and incrementation of a player's score. This quiz therefore allows to:

  • Create an online account (the identifiers are saved in the database (BDD))
  • Log in with a username, valid email address and password
  • Play the quiz with ephemeral session variables.
  • Saving the results in the database.
  • Logout

Accommodation

For hosting the quiz in Php, it is necessary to have a LAMP environment, because PHP is a language server oriented.
So I proceeded to the creation of a KVM virtual machine (Linux) then to the installation a LAMP environment on it (Linux Apache MySQL Php).


For what remains of the post, I have recreated the LAMP environment of my virtual machine on a machine hosted on AWS (Amazon Web Service). In theory, installing Lamp is now very easy, just enter:

$ sudo apt-get install lamp-server ^

Despite everything, several problems emerge when installing an SSL / TLS certificate, or phpmyadmin for example, so that it is configured WITH apache and has access to the mysql configuration files. In addition, the AWS platform has its own specificities such as the type of machine (Amazon Linux 2, Amazon AMI, ...) and the Firewall, the encryption key for connecting to the server via SSH, So we had to adapt to be able to install this working environment and have access to the files through the using the public address and a browser.

Finally, after a slow installation, all you have to do is upload all of my files with sftp, as well as the database by drag and drop on phpmyadmin.

Tree structure>

The tree structure of my files looks like this:

  • /
    • index.php (... home page ...)
  • jeu/ (... part of the site accessible only by members ...)
    • quizz.php (... game launch page with levels ...)
    • quest.php (... page with a question ...)
    • gagne.php (... page when the player won ...)
    • score.php (... page with the best scores ...)
  • php/
    • bd.inc.php (... DB configuration ...)
    • connexion.inc.php (... DB connection file ...)
    • mire.inc.php (... login pattern ...)
    • authorise.inc.php (... the user is logged in ...)
  • css/
    • quizz.css (... the CSS file ...)
  • img/ ( ... contains all images ...)

Database

The quiz database must meet several needs:

  • Authenticate users
  • Host the list of questions
  • Host the different answers, and annotate the correct one
  • Count the score of one or more players

To meet these needs, the database is structured as follows:
(* represent primary keys)

Player Questions Score
Login [VARCHAR 32] [PRIMARY] Identifier [VARCHAR 32] [PRIMARY] Identifier [VARCHAR 32] [PRIMARY]
Password [VARCHAR 32] Statement (question) [VARCHAR 256] Date [TIMESTAMP]
Email [VARCHAR 64] Response 1 [VARCHAR 32] Player [VARCHAR 32] [PRIMARY]
- Response 2 [VARCHAR 32] Score [INT]
- Response 3 [VARCHAR 32] -
- Response 4 [VARCHAR 32] -
- Correct answer number [INT] -
- Difficulty (1, 2 or 3) [INT] -
