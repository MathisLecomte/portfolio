February 26, 2021

Windows 10 comeback !

After changing the Partition Manager (RAID => AHCI), I realized without surprise that Windows 10 does not no longer demonstrated, the computer displayed a black screen on startup, then booted on another disk if I didn't have find windows 10 working.

As explained previously, my use of Windows 10 today is reduced to updating firmwares and my Dell-signed devices. On the other hand, having Windows 10 under the elbow and in fast start, to occasionally execute software exclusively for windows, for example, can be very practical.

Tools

This is where it comes into play KVM (Kernel-based Virtual Machine)


KVM is an Open Source virtualization technology integrated into linux that allows, to put it simply, launch virtual machines from the Kernel itself. This technique has the effect of making virtual machines very powerful since they use much better system resources than would Oracle VirtualBox for example.


I often use KVM to launch virtual machines allowing me to test a LAMP environment , MySQL database servers, or even to use Kali Linux for its Pen testing tools.

So I wanted to try to use Windows 10 with KVM, and my management CPU resources (Gnome extension)

To my surprise, Windows 10 runs smoothly. I obviously favor the installation of lightweight programs, but even video streaming on Zoom works.


Of course, having a modest graphics card (Intel UHD Graphics), this fluidity would quickly be implemented. shakes in the face of the use of greedy games or streaming in 4K. But these performances prove sufficient for my use.

-Operation and Installation

To create a KVM virtual machine, I use the virt-manager tool in GUI, but anything can be done in command line thanks to virsh and qemu .

In command line, this would give:

# virt-install --name=win10 \
--vcpus=4 \
--memory=16024 \
--cdrom=/tmp/windows10.iso \
--disk size=100 \
--os-variant=windows10

-Smoothing

The ideal in this business would be to be able to start Windows 10 just by pressing a button in my dock (my favorites bar) This is exactly what I did.

STEP 1:

Create a script that starts my virtual machine, and launches the Desktop Viewer.

start_windows10.sh
#! /bin/bash

virsh start win10
virt-viewer -cf qemu:///system win10

Then we grant the necessary permissions:

chmod +x ./start_windows.sh

STEP 2:

Create an icon in the applications menu

Again, there are different ways to create and edit icons in the app menu.

I could have gone to create a .desktop file in the ~/.local/share/applications directory, but I just used the software Alacarte for easily edit in graphic mode.
